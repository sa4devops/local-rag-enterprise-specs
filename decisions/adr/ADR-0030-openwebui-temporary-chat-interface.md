# ADR-0030 — Open WebUI واجهة محادثة مؤقتة (Interim Chat Client)

**السياق (Context):** بعد تنحية تجربة عميل Flutter ‏(AQL) نهائياً (مستودعها حذفه SA من GitHub قبل هذه الدفعة، ونسخة التجربة المحلية أُرشفت وحُذفت — موثق في تقرير الدفعة)، يبقى خط الواجهة المعتمد React/Vite قائماً بلا إعادة فتح، وتلزم واجهة محادثة **مؤقتة** جاهزة تجارياً للاستخدام البيني قبل اكتمال واجهة React — دون خلق مصدر حقيقة ثانٍ ودون مساس بأي قرار مقفل.

**الوضع السابق (Previous state):** الدستور م6 وADR-0002: ‏Open WebUI **ليس الواجهة النهائية** — مرجع دراسة فقط وأي استخدام داخلي مشروط بمراجعة قانونية؛ سجل التراخيص يحمل صف OpenWebUI ‏(v0.6.5) بوصفه **reference only — Conditional**؛ ‏D1/ADR-0017: ‏React هي الواجهة المعتمدة؛ لا يوجد أي قرار سابق يعتمد Flutter (تحقق بالجرد — صفر إشارة في الفرعين).

**القرار (Decision):**
1. يُعتمد **Open WebUI كواجهة محادثة مؤقتة (interim) حصراً**، عبر تكامل معزول في `local-rag-enterprise-platform/integrations/openwebui/` — ‏scaffold موثق أولاً، ولا تشغيل قبل استيفاء بواباته.
2. **الحدود القطعية:** ليست الواجهة الدائمة (React تبقى خط الأساس — لا يُعاد فتحه) · ليست مصدر حقيقة · **لا اتصال DB مباشراً** · لا تقرر صلاحيات (المنصة تعيد التحقق وتملك القرار — ADR-0029) · لا تملك Store/Workflows/Approvals/Audit الحقيقية · **لا اعتماد للمنصة على جداول Open WebUI الداخلية** (المحادثات تُعرَّف بمعرفات محايدة وكل إجراء منفّذ يسجَّل في المنصة فوراً).
3. **بوابات التفعيل (قبل أي تشغيل):** اعتماد صف صورة الحاوية في `decisions/license-review.md` (م6/م7) — قرار الترخيص خارج نطاق هذه الدفعة ويعود لمالك المشروع عند اعتماد واجهة التشغيل. · إصدار صورة مثبَّت بلا latest مع تسجيل digest حقيقي عند أول سحب · سياسات policies/ منفَّذة · لا white-label.
4. **شروط الخروج (Decommission):** الحد الأدنى من واجهة React البديلة متاح ثم تسلسل الخروج الموثق في `docs/DECOMMISSION_PLAN.md` (تشغيل متوازٍ → إيقاف الاستخدام الجديد → تصدير/هجرة → تعطيل → إبقاء التدقيق → إزالة باعتماد رسمي).
5. **معايير إعادة النظر في نقله لمستودع مستقل** (بنود مراقبة مستقبلية — لا فعل الآن): فريق مستقل يديره · دورة إصدار مستقلة فعلية · تعدد منتجات يستهلك التكامل ذاته.

**البدائل (Alternatives):** انتظار واجهة React دون واجهة بينية (مرفوض — يؤخر الاستخدام المبكر والتغذية الراجعة) · بناء واجهة محادثة مؤقتة من الصفر (مرفوض — كلفة بلا قيمة لمرحلة انتقالية) · إحياء عميل Flutter/AQL (مرفوض — التجربة أُنهيت ولم تكن قراراً معمارياً أصلاً) · ‏fork لـ Open WebUI أو نسخ كوده (ممنوع — D8/ADR-0021).

**السبب (Rationale):** واجهة جاهزة تُركَّب خلف عقود المنصة تعطي قيمة مبكرة بلا دين معماري، ما دامت محكومة بمبدأ العميل القابل للاستبدال وبخطة خروج مكتوبة قبل الدخول.

**العواقب (Consequences):** ‏scaffold معزول كامل التوثيق في المنصة (بلا تثبيت تشغيلي) · صف ترخيص جديد لصورة الحاوية في السجل الوحيد · دلتا محدودة في phase-roadmap تعلن المسار المؤقت وشروط خروجه دون تغيير أي مرحلة · اختبار الاستبدال (ADR-0029) يُكتب كمواصفة مؤجلة مربوطة بمرحلتها.

**خطة الانتقال (Migration plan):** الدخول: عبر بوابات التفعيل أعلاه فقط. الخروج: وفق `DECOMMISSION_PLAN.md` — بيانات المحادثات قابلة للتصدير/الهجرة، وسجلات التدقيق تبقى في المنصة (مصدر الحقيقة لم يغادرها أصلاً).
