# ADR-0022 — التطوير على GitHub + خط تسليم الشبكة المغلقة (بحد أدنى مغلق من Phase 0)

> **ملاحظة زرع وتحديث:** الرقم قائم في السجل الكنسي منذ 2026-07-04 (D7 — GitHub الآن + قناة نقل موقّعة؛ Git داخلي Backlog). يُزرع ملفه هنا ضمن إغلاق خط الأساس v1.0 **مع تفصيل تكميلي (D13+D14):** حد أدنى مغلق إلزامي من Phase 0 + **بوابة بروفة التسليم** قبل أول نشر إنتاجي مغلق.

**السياق (Context):** التطوير يجري online على GitHub بينما وجهة التشغيل النهائية شبكة مغلقة تماماً — يلزم خط تسليم موثوق قابل للتحقق وقابل للتراجع، وجاهزية مغلقة لا تُرحَّل كلها إلى نهاية المشروع.

**الوضع السابق (Previous state):** D7 وconstitution م19 والمنهجية §17 تنص على offline bundle موقّعة وlockfiles وdigest pinning؛ FR-0.16 يضع placeholders — دون قائمة حد أدنى مغلقة مسماة من Phase 0 ولا بوابة بروفة ملزمة.

**القرار (Decision):**
1. **GitHub هو مستودع التطوير** — لا Gitea/GitLab داخلي الآن؛ يُفتح هذا فقط إذا نشأ تطوير داخل الشبكة مستدام (ADR جديد).
2. **التسليم إلى الشبكة المغلقة عبر حزم إصدار/تحديث موثقة وموقّعة**.
3. **الحد الأدنى المغلق من Phase 0 (D14):** مستودع المصدر · lockfiles · Python wheelhouse/cache · pnpm offline store · صور Docker بـ digests · قوالب الإعداد · سكربت تثبيت · سكربت تحقق · checksums · integrity manifest · إشعارات التراخيص · model manifest متى وُجدت نماذج · تعليمات النسخ/الاستعادة · إجراء التحديث · إجراء التراجع.
4. **مؤجَّل قابل للإضافة (لا يُحذف موطؤه):** SBOM متقدم · بنية توقيع artifacts · مراقبة مؤسسية · تصليب سلسلة توريد متقدم.
5. **بوابة بروفة إلزامية (D13):** قبل **أول نشر إنتاجي داخل الشبكة المغلقة** تُنفَّذ بروفة موثقة كاملة لدورة: **بناء الحزمة → النقل → التحقق من checksums → التطبيق → التراجع**.

**البدائل (Alternatives):** Gitea/GitLab داخلي من الآن (مرفوض — عبء تشغيلي بلا مطوّرين داخل الشبكة بعد) · تأجيل كل عناصر الحزمة المغلقة إلى P8 (مرفوض — يفاجئ المشروع بديون نقلٍ متراكمة) · نقل يدوي غير موقّع (مرفوض — يكسر م19 وقابلية التحقق).

**السبب (Rationale):** الحد الأدنى المبكر يجعل «قابلية النقل» خاصية مبنية لا مهمة نهائية؛ والبروفة الموثقة تحوّل التسليم من افتراض إلى قدرة مثبتة قبل أن يترتب عليه إنتاج.

**العواقب (Consequences):** بنود D14 أهداف تسليم في Phase 0 (الكتلة الإضافية v1.0)؛ بوابة البروفة مضافة إلى phase-roadmap (منطقة الإنتاج)؛ تمرين النسخ/الاستعادة الكامل يبقى ضمن P8.

**خطة الانتقال (Migration plan):** عند نشوء تطوير داخلي مستدام: ADR لاستضافة Git داخلية + قناة مزامنة موقّعة ثنائية الاتجاه.
