# BACKLOG_DEFERRED_SCOPE.md — النطاق المؤجَّل بقرار

الحالة: Accepted v0.3 (2026-07-06) — وثيقة تخطيط ما-قبل-الأساس (استُعيدت 2026-07-10 ضمن إغلاق F-3). مرجع تاريخي معتمد لمدخلات التخطيط ومعرفات FR؛ ما استجد في v0.4 حتى v1.0-architecture-baseline والـ ADRs وphase-roadmap يعلوها عند أي تعارض (م20). لا تُستخدم للتنفيذ المباشر.

> **Version:** 1.0 — Proposed (Accepted باعتماد v0.3) · **Date:** 2026-07-06
> **قاعدة:** كل بند هنا **خارج مسار التنفيذ الأول (P0–P8)** عمداً؛ تفعيله لاحقاً يتطلب قرار مالك + ADR + بواباته (ترخيص/أمن/سعة). لا يجوز لأي وكيل سحب بند منه إلى مرحلة.

| البند | لماذا مؤجَّل | أبكر نقطة دخول منطقية | بوابات التفعيل |
|---|---|---|---|
| فهم الفيديو الكامل (multimodal video understanding) | كلفة عتاد/نماذج عالية؛ المسار الاحتياطي (صوت+keyframes) يغطي الحاجة | بعد P5 بثبوت حاجة | أوزان نماذج + قياس عتاد |
| فصل قواعد البيانات فيزيائياً / استخراج microservices | لا حاجة قبل أحمال حقيقية؛ التصميم microservice-ready أصلاً | بعد P8 بقياسات إنتاج | ADR + contract tests (Methodology §4) |
| K8s / OpenShift manifests | Compose يكفي dev/staging؛ الانتقال لاحقاً **بلا تعديل كود** | مع توسّع تشغيلي بعد P8 | قرار Ops + منصة الحاويات |
| استضافة Git داخلية (Gitea/GitLab) | لا فريق تطوير داخل الشبكة الآن (D7/ADR-0022) | عند نشوء تطوير داخلي | صف ترخيص + قرار تشغيلي |
| استراتيجية النشر العام/open-source والترخيص | المشروع خاص حالياً؛ الفصل specs/platform يحفظ الخيار | قرار ملكية مستقل | قرار قانوني/ترخيص كامل |
| تطبيقات أدوات أمنية في المتجر (نمط PentAGI/RedAmon لفرق SOC/CERT/Red Team) | حساسية عالية؛ تتطلب حوكمة P7 مكتملة وقراراً أمنياً صريحاً | بعد P7 | مراجعة أمن/ترخيص لكل أداة + موافقة قيادية |
| SSO/SAML/فيدرالية موسّعة | Local + IdP-adapter يكفيان المسار الأول | بعد P1 عند طلب المؤسسة | مكوّن الربط بصف ترخيص |

| Graph analytics متقدم (تفعيل AGE/Neo4j) | Optional-off؛ لا حاجة مثبتة بعد | عند حاجة علاقات معقدة (بعد P5) | فئة 3 + (Legal لـ Neo4j) |
| Observability stack كامل (Grafana/Loki/OTel-server) إن لم يُعتمد في P8 | الخط الأساسي يغطي وظيفياً؛ القرار قانوني أولاً | P8/OD-P8-1 | Legal + فئة 3 |
| قنوات إشعار خارجية (Email/SMS) | in-app يكفي المسار الأول؛ الطبقة مجرّدة | بعد P3 عند توفر بوابات داخلية | صفوف مزوّدات + أمن |
| Watermarking/DLP متقدم للتصدير | P4 يغطي التصدير المحكوم الأساسي | بعد P4 بطلب | أدوات بصفوف ترخيص |
| حقول محسوبة server-side معقدة (OD-P2-2) | خطر تعقيد مبكر | بعد P4 | تصميم مصغّر + اعتماد |
| SLA/تصعيد متقدم للworkflows | الأساسي (تذكيرات استحقاق) يكفي أولاً | بعد P3 | تصميم مصغّر |
