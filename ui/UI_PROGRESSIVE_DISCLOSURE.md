## 1) هرم الإظهار (المستويات الأربعة — Stitch يلتزم بترتيبها)
| المستوى | ماذا يظهر | كم % من المحتوى الكلي | يُفتَح بـ |
|---|---|---|---|
| **L0 — Essentials** | القرار الرئيسي + هويّة العنصر + الحالة الحاكمة | ≤30% | فوري (default) |
| **L1 — On-demand summary** | ملخص السياق المصاحب | ~30% إضافي | نقر «مزيد»/فتح تبويب/توسّع inline |
| **L2 — Detail surface** | التفاصيل الكاملة | ~30% إضافي | drawer/modal/tab |
| **L3 — Full Renderer / dedicated page** | كل التفاصيل + كل الأفعال + السياق التاريخي | البقية | زر «فتح كامل» ⇒ page |

**قاعدة الحسم:** إن لم يخدم العنصر L0، فمكانه L1 على الأقل — لا في السطح الأول.

## 2) قواعد اختيار مستوى الإظهار (شجرة قرار)
1. هل يحتاجه المستخدم **لاتخاذ القرار الأساسي على هذه الشاشة**؟ ⇒ L0.
2. هل يحتاجه **لفهم السياق قبل القرار** لكن ليس دائماً؟ ⇒ L1 (مطويّ عنوانياً).
3. هل هو **تفاصيل تحرير/مقارنة/عرض موسّع**؟ ⇒ L2 (drawer/modal/tab).
4. هل هو **مهمة طويلة أو عرض كامل بروابط مرجعية**؟ ⇒ L3 (page/Renderer).
5. إن لم يجد المصمم مكاناً واضحاً ⇒ العنصر **لا يستحق الوجود على هذه الشاشة**.

## 3) قواعد صارمة ضد الازدحام (Stitch يلتزم حرفياً)
1. **عنوان واحد h1 لكل صفحة**. لا h1 متعدد.
2. **حد أقصى 3 أزرار أفعال مرئية** في الرأس/الشريط العلوي — الباقي في قائمة «مزيد» (⋯) — تُبنى من Action Registry بترتيب priority.
3. **حد أقصى 5 مؤشرات/شارات** في header العنصر (تصنيف · workflow state · اعتماد · مخاطر · مُولَّد — انتهى).
4. **الجداول: 5–7 أعمدة ظاهرة كحد أقصى**، الباقي في **column picker** أو drawer التفاصيل.
5. **النماذج: أقسام قابلة للطي** — قسم واحد مفتوح افتراضياً (الأساسي)، البقية مطوية.
6. **panels: تبويب واحد مفتوح افتراضياً**، البقية مطوية عنوانياً.
7. **لا «كل شيء ظاهر افتراضياً»** — القاعدة معكوسة: الافتراضي مطويّ إلا استثناءً مذكوراً بقاعدة.
8. **لا كتل نص طويلة** — أكثر من ثلاثة أسطر ⇒ «اقرأ المزيد» (inline expand).
9. **لا تكرار للأزرار** — الزر الذي يظهر في الرأس لا يُعاد في التذييل.

## 4) متى كل سطح؟ (ملخّص عملي مرتبط بـ Interaction Model §4)
| الحاجة | السطح المفضّل | لماذا |
|---|---|---|
| تبديل عرض لنفس البيانات | **tab** | خفيف، يحفظ الإطار |
| قراءة موازية مع الأصل | **drawer end** | يظل الأصل مرئياً جزئياً |
| قرار قصير حرِج | **modal** | حاجب مركّز |
| مهمة طويلة أو مرجعية | **page** | bookmark-able، رجوع بالمتصفح |
| توسّع تفاصيل الصف | **inline expand** | لا يكسر السياق الجدولي |
| مصاحب دائم للشاشة | **panel** | جزء من التخطيط |

**تحذيران:** (١) لا تستخدم modal حيث drawer يكفي (يعطّل الباقي بلا داعٍ). (٢) لا تستخدم page حيث drawer يكفي (يفقد السياق).

## 5) تطبيق على الأسطح الرئيسية

### 5.1 AI Workspace (`ws.main`)
- **L0 (دائم):** المحادثة + composer + شارة «مُولَّد» على مخرَج النموذج + شريط الجلسات المصغّر.
- **L1 (مطويّ عنوانياً):** context panel — Citations · Related · Attachments؛ يفتح تبويب واحد تلقائياً عند وصول محتوى جديد.
- **L1 مشروط:** `panel.audit` (رأس التبويب يظهر للمخوّل فقط، محتواه مطويّ).
- **L2 (drawer):** تفاصيل استشهاد، معاينة سجل، تفاصيل تدقيق — drawer end واحد في كل مرة.
- **L2 (modal):** تأكيد typed/dual، معاينة HITL — لا شيء غير ذلك.
- **L3 (page):** «عرض الكل في القائمة» ⇒ `run.list`، «فتح كامل» ⇒ `run.detail`، «فتح المصدر» ⇒ صفحة المصدر إن وُجدت.
- **قاعدة لـ Stitch:** لا تعرض كل الاستشهادات مفتوحة؛ عنوان + عدد + توسّع.

### 5.2 Action Cards (داخل ws.thread)
- **L0:** رأس البطاقة (اسم الفعل + مخاطر + تصنيف) + السطر التنفيذي «سيتم…» + الحقول **الإلزامية فقط** + زر التأكيد.
- **L1 (inline expand):** الحقول الاختيارية مطوية خلف «مزيد»؛ الأثر التفصيلي/التبعيات مطوية.
- **L2 (drawer):** «معاينة السجل المرتبط» — drawer end.
- **L3 (page):** «فتح النموذج الكامل» ⇒ `run.form` بحقول مملوءة مسبقاً.
- **قاعدة لـ Stitch:** بطاقة action_preview لا تعرض >7 حقول افتراضياً؛ الزائد خلف «مزيد».

### 5.3 Runtime Renderer (`run.list` · `run.form` · `run.detail`)
- **`run.list` L0:** 5–7 أعمدة، شريط فلترة معلن مطويّ (يفتح على طلب)، بحث سريع، صف واحد مختار.
  **L2:** فتح الصف = drawer end بمعاينة؛ «فتح كامل» ⇒ page `run.detail`.
  **قاعدة:** الأعمدة الإضافية عبر column picker، ليست كلها معاً.
- **`run.form` L0:** القسم الأساسي مفتوح؛ الأقسام الأخرى (الحوكمة، المرفقات، السياسات) مطوية.
  **L1:** كل قسم يفتح عند النقر، يحمل حالته الخاصة (خطأ/سليم).
  **قاعدة:** لا تعرض كل الأقسام مفتوحة أبداً.
- **`run.detail` L0:** شارة الحالة + بيانات أساسية + **حتى 3 أزرار أفعال** في رأس الصفحة.
  **L1 مطويّ:** بقية الأفعال في «مزيد»؛ لوحات الحقول الإضافية في tabs.
  **L1 مشروط:** `panel.audit` جانبية مطوية للمخوّلين.
  **قاعدة:** لا تعرض عشرات الأزرار — ثلاثة رئيسية والباقي مجمَّع.

### 5.4 Work Queue (`queue.tasks` · `queue.approvals`)
- **L0:** قائمة بأولوية + عدد المهام الفاعلة + فلتر «مسندة إليّ اليوم» افتراضياً.
- **L1:** فلاتر متقدمة مطوية (حالة/تاريخ/مشروع)؛ درج التفاصيل عبر نقر الصف.
- **L2 (drawer/page):** بند المهمة = drawer end؛ **بند الاعتماد = page مستقلة `queue.approval_detail`** لأن القرار رسمي.
- **قاعدة:** لا تعرض threads داخل القائمة — التفاصيل في السطح المناسب.

### 5.5 Admin Console
- **`admin.dashboard` L0:** ≤4 بلاطات (اعتمادات معلّقة، مهام فاشلة، صحة، آخر تغييرات سياسات) + بحث موحّد.
- **`admin.record_types` L0:** قائمة الأنواع + زر «تعريف جديد»؛ الـ Builder صفحة مستقلة.
- **Builder (`page`) L0:** شريط دورة الحياة + التبويبات (الحقول · السياسات · التنقّل)، تبويب واحد نشط.
  **L1:** diff عبر drawer end؛ محاكاة «مَن سيرى؟» في drawer end.
- **قاعدة:** لا تعرض «كل الحقول + كل السياسات + كل النسخ» على شاشة واحدة.

### 5.6 RAG / Knowledge Admin
- **`admin.rag_sources` L0:** المصادر + النطاق + الحالة.
  **L2 (drawer):** تفاصيل المصدر + إعادة الفهرسة.
- **`admin.ingestion` L0:** طابور المهام الأخيرة مع الحالة (نجاح/فشل/معالجة) وعدد.
  **L1 (inline expand):** خطوات الملف الواحد؛ **L2 (drawer):** تفاصيل خطوة مع error_code.
- **`admin.retrieval_tester` L0:** حقل استعلام + زر تشغيل.
  **L1:** النتائج تظهر بعد التشغيل مع لافتة قرار الكفاية.
  **L2 (drawer):** تفصيل provenance لكل نتيجة.
- **قاعدة:** لا تُظهر «كل السجل» في الصفحة نفسها — الأحدث فقط + رابط للأرشيف.

### 5.7 Provider/Model Registry (`admin.capabilities` · `admin.agents`)
- **L0:** جدول capability→provider/model + fallback + حالة.
  **L2 (modal):** تغيير الربط (خطوات محدودة، تأكيد typed، شارة «يتطلب اعتماداً في الإنتاج»).
  **L2 (drawer):** خطة إعادة الفهرسة قبل تأكيد تغيير embedding.
- **قاعدة:** لا endpoints/secrets/GPU أبداً — لا في L0 ولا في أي L آخر.

### 5.8 Audit / Ops
- **`admin.audit` L0:** فلاتر أساسية (فاعل/فعل/تاريخ) + قائمة.
  **L2 (drawer):** تفصيل الحدث + سلسلة correlation.
  **L3 (page):** تصدير محكوم = صفحة الحالة/التحميل.
- **`admin.health` L0:** بلاطات فعّال/مطفأ مع فرز الأخطاء أولاً.
  **L1 (inline expand):** تفاصيل التبعية الواحدة.
- **`ops.status` L0:** ملخص + آخر نسخة احتياطية + صحة الموصلات؛ **قراءة فقط بلا أزرار lifecycle**.

## 6) قواعد للنماذج والجداول (تفصيل — لأنها الأكثر ازدحاماً)
### النماذج
- قسم رئيسي مفتوح؛ ما بعده مطويّ عنوانياً.
- الحقول المشروطة (visibility conditions) لا تحتلّ مساحة قبل تفعّل شرطها.
- الحقول المتقدمة/الحوكمية (تصنيف، نطاق، مرفقات) في قسم منفصل غير الأساسي.
- أخطاء التحقق تُثبَّت على الحقل مع رسالة i18n مقتضبة؛ ملخص الأخطاء أعلى الصفحة بروابط قافزة.

### الجداول
- 5–7 أعمدة ظاهرة؛ column picker مطويّ لبقية الأعمدة.
- فلاتر متقدمة في drawer end (لا تحتلّ الرأس).
- فرز أحادي افتراضاً؛ متعدد خلف تفعيل صريح.
- **row actions** مجمَّعة تحت ⋯ إذا زادت عن 3؛ الأكثر استخداماً فقط ظاهرة.

## 7) قاعدة إرسال المستخدم إلى Runtime Renderer (متى تُغادر البطاقة/الدرج)
| المؤشر | الاستجابة |
|---|---|
| نتيجة >10 صفوف | «عرض الكل في القائمة» ⇒ `run.list` |
| تحرير >7 حقول | «فتح النموذج الكامل» ⇒ `run.form` |
| مقارنة نُسخ سجل / سلسلة تدقيق طويلة | ⇒ `run.detail` أو `admin.audit` |
| عمل مستمر يحتاج bookmark | ⇒ page مباشر |
| مهمة تستغرق >90 ثانية | ⇒ تحويل لمهمة غير متزامنة + إشعار عند الاكتمال |

## 8) قاعدة لـ Stitch (تُلصق في كل prompt)
> **قواعد الإظهار التدريجي الملزمة:** ابدأ بـ L0 فقط. اعرض ≤3 أزرار في الرأس، ≤5 شارات، ≤7 حقول في نموذج/بطاقة، ≤7 أعمدة في جدول. كل ما عدا ذلك مطويّ خلف «مزيد»/tab/drawer وفق قواعد الأسطح. لا تجمع كل الخصائص على شاشة واحدة؛ إذا شعرت أن الشاشة ممتلئة ⇒ انقل إلى drawer أو Renderer.
