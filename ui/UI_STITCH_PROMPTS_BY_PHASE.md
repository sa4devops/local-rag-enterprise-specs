## §0 طريقة الاستخدام

## §1 الديباجة المشتركة (تُلصق حرفياً قبل كل مجموعة)
> Design language: “Calm Institutional Command” — a formal governmental enterprise system. Surfaces near-white (#FFFFFF / #F4F6F9), ink #1B2430, single leading color deep institutional blue #1F4E79. RESERVED palettes: 5-level data-classification chips (gray #6B7280, blue #2F6FAD, amber #B7791F, orange #C2410C, dark red #8B1E1E — used ONLY for classification, always icon+text+color) and violet #6D5BD0 used ONLY for an “AI-generated / مُولَّد” badge. Typography Arabic-first (IBM Plex Sans Arabic style), Latin digits in data. **Primary direction RTL with Arabic labels; English is secondary.** Generous whitespace, 6–10px radii, hairline borders, two soft elevations, no gradients, no emojis, no consumer-app playfulness, WCAG AA. Use realistic **fictitious** Arabic enterprise sample data (invented ministry-style names). Do NOT imitate or copy branding, logos, icons, or layouts of any existing product (Open WebUI, ChatGPT, Slack, Notion, Appsmith, Budibase, Directus, NocoDB, Dify, Microsoft, Linear, GitHub, or others). Output is a **static visual prototype for reference only; its code will not be used** — final implementation is a separate governed build.

## G1 — Global Design System (المرحلة: أساس P0)
**الهدف:** لوحة أسس مرئية تُلزم كل المجموعات. **الشاشات:** صفحة style-guide واحدة. **المكوّنات:** لوح الألوان (مع لوحتي الحجز: التصنيف/مُولَّد) · سلم الطباعة عربي/لاتيني · الأزرار (primary/secondary/tertiary/danger + متغيّرا التأكيد typed/dual) · **تصنيف الشارات كاملاً** (دورة حياة/اعتماد/مخاطر/تصنيف/مُولَّد/مرجع تدقيق) · جدول وعناصر نموذج ونافذة ولافتات (بيئة/Safe Mode/model-down) · حالات فارغ/تحميل skeleton/خطأ (error_code+request_id). **الحالات:** الافتراضي+hover+disabled-مع-سبب. **RTL:** كل العينات RTL أولاً + مثال عزل LTR لمعرّف/SQL داخل نص عربي.
> Create a single style-guide canvas implementing the shared preamble: color tokens (including the two RESERVED palettes), Arabic-first type scale, button hierarchy with “typed” and “dual (countdown)” confirmation variants, the full badge taxonomy (lifecycle Draft→Active→RolledBack; approval Pending/Approved/Rejected/Returned; risk low→critical; 5 classification chips icon+text; violet “مُولَّد” badge; copyable audit-ref chip), sample table/form/modal/banner, and empty/skeleton/error states. Everything laid out RTL with Arabic labels; include one example of an LTR-isolated identifier inside Arabic text. No product imitation; static reference only.

## G2 — Auth (P1)
**الشاشات:** auth.login · auth.first_login. **المكوّنات/الحالات:** بطاقة دخول مركزية رسمية بلا صور تسويقية · حالات خطأ اعتماد/حساب مقفل/جلسة منتهية داخل الشاشة · فرض تغيير كلمة بأول دخول (قواعد القوة كتلميحات) · شريط بيئة غير-إنتاج. **RTL:** حقول ولافتات RTL؛ اسم المستخدم ASCII بعزل LTR. **لا تفعل:** لا تسجيل اجتماعي، لا «استعادة عبر البريد».
> Design a formal governmental login (auth.login) and forced first-login password change (auth.first_login): centered card, Arabic-first RTL fields, inline states for bad credentials / locked account / expired session, non-production amber environment strip on top, password-policy hints. No marketing hero, no social login, no imitation of any product. Static reference only.

## G3 — Enterprise AI Workspace Shell (P6) — **ليست chatbot**
**الشاشات:** ws.main. **المكوّنات:** بنية ثلاثية (session rail / thread / context panel بتبويبات: Citations · Related · Attachments · Audit-مخوّل) · مُنشئ برسالة + إرفاق + **Scope selector** (المسموح فقط) · رسائل بلا فقاعات ملونة (أسطر محاذاة بفواصل رفيعة) · شارة «مُولَّد» على كل دور نموذج · لافتات model_down/Safe-Mode. **الحالات:** فارغة أولى (قدرات المستخدم الفعلية) · بث/نبض · **الوضع الحتمي كامل الشاشة** (بحث كلمات + مشغّل أفعال، والرسالة: «كل العمليات متاحة عبر القوائم»). **لا تفعل:** أي مظهر messaging/consumer chat، لا أفاتارات لعوبة، لا تنفيذ مباشر من رسالة.
> Design ws.main as an **enterprise command workspace, explicitly not a chat app**: three-zone layout (start-side session rail; central thread rendered as aligned formal rows with hairline separators — no colored bubbles; end-side context panel with tabs Citations / Related records / Attachments / Audit trace [visibly permission-gated]). Composer with attach button and a “knowledge scope” selector showing only permitted sources. Violet “مُولَّد” badge on every AI turn. Include: first-empty state listing this user’s actual capabilities, streaming state, model-unavailable banner with a full **deterministic mode** variant (keyword search + an action launcher palette; note “all operations remain available via menus”). RTL Arabic-first. Static reference only; no product imitation.

## G4 — Action Cards &amp; Governed Results (P6)
**الشاشات/المكوّنات:** داخل ws.main: `card.action_preview` (رأس: اسم الفعل + chip مخاطرة + chip تصنيف · جسم: شبكة حقول قابلة للتحرير بأخطاء خادمية مثبتة · تذييل: سطر الأثر المعلن + تحذير اللارجعة + أزرار تأكيد simple/**typed**) · شارة PENDING_APPROVAL بتتبع · EXECUTING · `card.result` (روابط فتح السجل + **مرجع تدقيق #** + تراجع-إن-مدعوماً) · بطاقتا الرفض (صلاحية بدرع / **No-Guessing** بأدلة: ما بُحث ولماذا لا يكفي + فعلان آمنان) · عنصر الاستشهاد (عنوان/محدد صفحة-مقطع-طابع/تصنيف/منشأ/فتح) · `card.sql_answer` (نص الاستعلام بخط mono **LTR معزول** + نسخ + جدول نتيجة) · بطاقة نتائج كبيرة بزر «عرض الكل في القائمة». **الحالات:** لوحة قصصية S3→S9 كاملة.
> Design the governed action-card system inside the workspace: an editable **preview card** (header: action name + risk chip + classification chip; body: field grid with inline server-side validation errors; footer: a declared-effect sentence, an irreversible warning when applicable, and confirm buttons including a “typed confirmation” variant), then states: pending-approval chip with tracking, executing, success **result card** with open-record links, copyable audit-ref, and optional Undo; two refusal cards — permission-denied (shield) and insufficient-evidence “No-Guessing” (shows what scopes were searched, why evidence is insufficient, plus two safe next actions); a citations panel item (title, precise locator, classification chip, origin tag, open button); an SQL-answer panel with the query always visible in LTR-isolated mono + copy + result table; and a large-result summary card with an “Open full list” handoff button. Present as an S3→S9 storyboard. RTL Arabic-first; static reference only.

## G5 — Admin Console Shell + Platform Config (P1 · P4)
**الشاشات:** صدفة الـ Console · admin.dashboard · admin.features. **المكوّنات:** sidebar مجموعات مرشّحة بالصلاحية · breadcrumbs · بحث إداري · شريط بيئة · **مظهر Control Plane**: شريط نسخة/حالة أعلى التعريفات · dashboard بمؤشرات دنيا (اعتمادات معلّقة/مهام فاشلة/ملخص صحة) · features: قائمة مزايا + **لوحة تبعيات** + عناصر A10 رمادية مقفلة + لافتة Safe Mode + حوار أثر التبعيات بتأكيد typed. **لا تفعل:** لا يظهر أي endpoint/secret.
> Design the admin console shell and platform-config screens as a **control plane**: permission-filtered grouped sidebar, breadcrumbs, admin search, environment strip; a minimal dashboard (pending approvals, failed jobs, health summary); a feature-management screen with a dependency-impact side panel, permanently locked grey “protected core” items, an active Safe-Mode banner, and a typed-confirmation impact dialog. Formal, version/status bars visible on definitions. RTL Arabic-first; no secrets or endpoints anywhere; static reference only.

## G6 — Users · Org · Roles · Permissions · Policies (P1 → P4)
**الشاشات:** admin.users/user_detail/org/roles/permissions/policies/classification/security_policies. **المكوّنات:** list+detail · حقول ثوابت **مقفلة** بأيقونة · شجرة تنظيمية · مصفوفة أدوار×صلاحيات · **Policy Studio**: بانٍ شرطي مُهيكل (سمة/عامل/قيمة صفوفاً — **لا كود حر**) + **لوحة محاكاة «ماذا يرى X؟»** بنتيجة مفسَّرة موسومة «محاكاة — لا أثر» · شاشة تصنيف بدرجاته الخمس · سياسات أمن (جلسات/قفل/تصدير). **الحالات:** فارغ/بحث/SoD-تعارض مرفوض.
> Design security &amp; access screens in control-plane style: users list + user detail (immutable identity fields shown locked, roles, relationship grants with expiry, a small “last permission decisions” explainer), org tree, roles + permission matrix, and a **Policy Studio** with a structured condition builder (attribute / operator / value rows — no free code) plus a simulation side panel “what does user X see?” returning an explained, clearly-labelled no-effect result; classification levels screen using the five reserved chips; session/lockout/export security policies. Include an SoD-conflict rejected state. RTL Arabic-first; static reference only.

## G7 — Metadata · Actions · Migrations (Control Plane) (P1 → P3) — **ليست low-code للمستخدم**
**الشاشات:** admin.record_types(+fields tab) · admin.actions · admin.migrations · admin.generation_docs · admin.navigation. **المكوّنات:** Builder بشريط دورة حياة Draft→Validated→Approved→Active + تبويبا الحقول والسياسات (RLS نطاق/FLS مصفوفة) + **عرض diff** بين النسخ · **محرر عقد الفعل** (حقول العقد الـ13 كنموذج مُهيكل + محاكاة «مَن سيرى هذا الزر؟») · **مراجعة الترحيلات**: الاقتراح + تقرير مخاطر + اعتماد **typed** بشخص مختلف (SoD ظاهر) · توثيق التوليد للقراءة · محرر تنقّل بصلاحية لكل عنصر.
> Design the metadata/actions/migrations configuration area strictly as an **admin control plane (not an end-user app builder)**: a record-type builder with a definition-lifecycle bar (Draft→Validated→Approved→Active), fields tab, RLS-scope + FLS-matrix policy tab, and a neutral version-diff view; an **action-contract editor** presenting the 13-field governed contract as a structured form with a “who will see this button?” simulation; a **migration review** screen showing the proposal, a risk report, and a typed approval by a different approver (separation-of-duties visibly enforced); a read-only generation-documentation view; and a navigation editor with per-item permission. RTL Arabic-first; static reference only.

## G8 — Work Queue · Approvals · Workflow Config (P3)
**الشاشات:** queue.tasks · queue.approvals · queue.approval_detail · notif.center · projects.main · admin.workflows/workflow_detail. **المكوّنات:** مهامي (قائمة+درج تفاصيل) · صندوق اعتمادات · **سطح قرار رسمي**: شريط موافقة/رفض/إرجاع + تعليق إلزامي عند الرفض + **thread نقاش** لأطراف البند · مركز إشعارات + threads · مشاريع master-detail · تعريف workflow كجدول حالات/انتقالات مُهيكل + لوحة Validation (دورات/حالات ميتة) + شريط دورة الحياة + تبويب مصفوفة الاعتماد. **الحالات:** بند HITL (P7) يظهر لاحقاً بنفس القالب — اترك متغيّراً له.
> Design the work-queue and workflow-governance set: My Tasks (list + detail drawer), an approvals inbox, and a **formal approval-decision surface** (approve / reject / return bar, mandatory comment on reject, a participants-only discussion thread, decision receipt with audit-ref); a notifications center with per-item threads; projects master-detail; and workflow configuration as a structured states/transitions table with a validation panel (cycles / dead states), the definition-lifecycle bar, and an approvals-matrix tab. Include a visual variant of the same approval surface for an external-action (human-in-the-loop) payload preview. RTL Arabic-first; static reference only.

## G9 — Knowledge · Ingestion · Retrieval Tester (P5)
**الشاشات:** admin.rag_sources · admin.ingestion · files.manager · admin.okf · **admin.retrieval_tester**. **المكوّنات:** مصادر بنطاقات = وسوم صلاحية · مراقب خط المعالجة (خطوات لكل ملف بحالة/error_code/إعادة محاولة) · إدارة ملفات بشارات تصنيف وحوار إعادة تصنيف بالأثر · قائمة OKF موسومة «ليست مصدر حقيقة» · **مختبر الاسترجاع**: استعلام تجريبي + نتائج بمحددات provenance دقيقة + **لافتة قرار الكفاية** (كافٍ/غير كافٍ) — موسوم «أدمن — لا يتجاوز صلاحيات المُجرِّب».
> Design the knowledge-administration set: RAG sources where each source scope is shown as permission tags; an ingestion monitor with per-file pipeline steps (status, error code, retry); a files governance screen with classification chips and a reclassify dialog showing impact; an OKF bundle list labelled “not a source of truth”; and an admin **retrieval tester** — a test query box returning results with precise provenance locators plus an evidence-sufficiency verdict banner (sufficient / insufficient), clearly marked as running within the tester’s own permissions. RTL Arabic-first; static reference only.

## G10 — Provider / Model Registry (P5 → P6)
**الشاشات:** admin.capabilities · admin.agents. **المكوّنات:** جدول capability→provider/model + fallback · **نتيجة اختبار اتصال chip فقط** (نجاح/فشل — بلا endpoint) · شارة «يتطلب اعتماداً في الإنتاج» · **حوار خطة إعادة الفهرسة** عند تغيير embedding · ربط لكل وكيل بمعاملاته وfallback. **لا تفعل:** لا endpoints/secrets/GPU إطلاقاً (D9).
> Design the model/provider registry as a governed control plane: a capabilities table mapping each capability to provider/model with an explicit fallback column; a test-connection action that returns only a success/fail chip (never endpoints or secrets); a visible “approval required in production” badge on binding changes; a re-index-plan warning dialog when the embedding model changes; and a per-agent bindings screen (model, parameters, fallback). RTL Arabic-first; static reference only.

## G11 — Integrations · App Store · Connections (P7)
**الشاشات:** store.catalog · store.connections · admin.connectors. **المكوّنات:** بطاقات متجر ثنائية اللغة مرشّحة بالصلاحية بحالة الربط · ارتباطاتي (تجديد/إلغاء — الرموز لا تُعرض) · إدارة موصلات: أدوات بوسمي read/act ومستوى خطورة + محدد جمهور + سياسة HITL لكل أداة + chip صحة الموصل · **حوار HITL** (payload مقروء قبل الموافقة) بنفس قالب G8. **لا تفعل:** لا أسرار، لا شعارات منتجات حقيقية (أسماء موصلات وهمية).
> Design the integrations set: a permission-filtered bilingual app-store catalog with link-state on each card; a “my connections” screen (renew / revoke; tokens never displayed); a connector administration screen listing tools tagged read/act with risk level, an audience selector, per-tool human-in-the-loop policy, and a connector-health chip; plus the HITL approval dialog showing a readable action payload before consent (reusing the formal approval surface). Use fictitious connector names; no real product logos, no secrets anywhere. RTL Arabic-first; static reference only.

## G12 — Audit · Logs · Health · Ops (P1 → P8)
**الشاشات:** admin.audit · admin.logs · admin.health · admin.error_catalog · admin.config_profiles(ro) · ops.status · about.licenses. **المكوّنات:** عارض تدقيق (فلاتر فاعل/فعل/كيان/تاريخ + إشعار «ضمن نطاقك» + لوحة تفصيل بسلسلة correlation) · سجلات بقناع أسرار · صحة تُظهر **المطفأ disabled بوضوح** · كتالوج أخطاء · إعدادات **قراءة فقط** بنمط D9 (قفل دائم) · سطح تشغيل قراءة (نسخ احتياطية/إصدارات/صحة موصلات) · صفحة Attribution.
> Design the observability and operations set: an audit viewer (actor/action/entity/date filters, a visible “scoped to your permissions” notice, detail side panel showing the correlation chain from proposal to execution); an application-log viewer with masked secrets; a health page clearly distinguishing enabled vs disabled components; an error-code catalog; a read-only configuration/profiles view styled with a permanent lock (no edit affordances at all); a read-only ops status surface (backups, versions, connector health, runbook links); and an About/Attribution page listing third-party licenses. RTL Arabic-first; static reference only.

## G13 — Runtime Renderer Templates (P1 → P2) — **قوالب لا شاشات كيانات**
**الشاشات:** run.list · run.form · run.detail · profile.entity. **المكوّنات:** قائمة (شريط فلترة معلن + مبدّل كثافة + أعمدة RTL + خلايا FLS مقنَّعة + أفعال صف) · نموذج (أقسام، تحققات خادمية، حقل مرفق [متغيّر P5]) · تفاصيل (**صف أزرار من الـ Registry** مع مثال زر معطَّل-بسبب + لوحة تدقيق جانبية مخوّلة + شارات تصنيف) · ملف كيان بخط زمني. **إثبات العمومية:** اعرض **نفس القالب بكيانين وهميين مختلفين**.
> Design the four deterministic runtime **templates** (not entity-specific apps): a data list (declared filter bar, density toggle, RTL column order, masked cells for hidden fields, row actions), a record form (sections, server-side validation states, an attachment field variant), a record detail (an action-buttons row driven by a registry — include one disabled-with-reason button — a permission-gated audit side panel, classification chips), and an entity-profile with a timeline. **Prove genericity by showing the same templates skinned with two different fictitious entities.** RTL Arabic-first; static reference only.
