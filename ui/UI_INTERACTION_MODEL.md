## 1) المصطلحات (تعريف واحد لا يقبل الالتباس)
| السطح | التعريف الدقيق | يتحمّل حالة URL؟ | ينافس المحتوى الرئيسي؟ | الوقت المتوقع |
|---|---|---|---|---|
| **page** | صفحة كاملة بمسار route ثابت | نعم (bookmark-able) | يحلّ محله | مهمة طويلة/رسمية |
| **tab** | تبديل داخل نفس الصفحة يحافظ على الإطار العام | نعم (سلسلة استعلام) | تبديل، لا يحلّ محل | ≤دقيقة تصفح |
| **drawer** | لوحة جانبية start/end تنزلق فوق أو بجانب المحتوى | جزئياً (اختياري) | يوازيه (لا يحجبه كلياً) | ≤دقيقتين قراءة/تفاعل خفيف |
| **modal** | نافذة مركزية حاجبة | لا | يحجبه كلياً | قرار حرِج قصير |
| **panel** | لوحة سياق ثابتة داخل الشاشة (ليست انزلاقية) | لا | جزء دائم من التخطيط | مصاحب طوال الجلسة |
| **inline** | توسّع/طيّ داخل الصف/البطاقة نفسها | لا | لا | ≤10 ثوان |

**قاعدة الإقصاء:** لا يُفتح modal فوق modal؛ فتح drawer يغلق drawer سابقاً؛ فتح modal يجمّد كل الـ drawers والـ panels خلفه.

## 2) قواعد اختيار السطح (شجرة قرار مُلزمة لـ Stitch)
1. هل يحتاج المستخدم **إشارة مرجعية / رابطاً / رجوعاً بالمتصفح**؟ ⇒ **page** (وإلا لا).
2. هل يحل محل المحتوى الرئيسي **بالكامل** ولمدة طويلة؟ ⇒ **page**.
3. هل هو محتوى مواز يريد المستخدم رؤيته **جنباً إلى جنب** مع الأصل؟ ⇒ **drawer** (start=سياقي، end=تفصيلي).
4. هل هو قرار **قصير حرِج** يوقف كل ما سواه؟ ⇒ **modal** (تأكيد/رفض/موافقة صريحة).
5. هل هو **تبديل عرض** لنفس البيانات؟ ⇒ **tab**.
6. هل هو **حقل تفصيلي فرعي** للصف/البطاقة نفسها؟ ⇒ **inline** (توسّع/طيّ).
7. هل هو محتوى سياقي دائم يرافق كل الشاشة؟ ⇒ **panel** (ثابت لا انزلاقي).

**قاعدة عدم التكرار:** أي محتوى يظهر في drawer لا يُعاد كـ modal ولا يُكرَّر page — تختار المشروع سطحاً واحداً لكل غرض.

## 3) وضع الأسطح الافتراضي (default = مطويّ إلا استثناءً)
| المستوى | الحالة الافتراضية | يفتح بـ |
|---|---|---|
| الإطار الرئيسي (topbar/sidebar) | ظاهر مضغوط (sidebar مطوي 72px تُفتح للمرور) | نقر/hover |
| تبويبات ضمن الصفحة | التبويب الأول فقط نشط | نقر التبويب |
| اللوحة اليمنى في ws.main | **panel.citations مطويّ عنوانياً** (رأس فقط) — يفتح عند وجود استشهاد جديد أو نقر المستخدم | نقر/حدث |
| panel.audit | **مطويّ دائماً** (يظهر رأسه فقط للمخوّل، وإذا لم يكن مخوّلاً فلا رأس) | نقر المخوّل |
| drawer التفاصيل داخل القوائم | مغلق | نقر صف |
| modal | مغلق | فعل صريح |
| inline expand | مطويّ | زر «مزيد»/سهم |

**قاعدة صارمة:** أي عنصر لا يخدم القرار الحالي **لا يظهر افتراضياً** — يُطوى أو يُخفى وفق §4.

## 4) مواضع الاستخدام لكل سطح (مصفوفة الحسم — Stitch يلتزم حرفياً)
| الحالة | السطح | مبرّر |
|---|---|---|
| فتح سجل من قائمة | **drawer end** أولاً (معاينة)؛ زر «فتح كامل» ⇒ page `run.detail` | حماية السياق الجدولي |
| تحرير سجل | **page** `run.form` (مسار new/edit) | مهمة طويلة، يحتاج bookmark ورجوع |
| تأكيد فعل typed/dual | **modal** | قرار قصير حرِج يوقف الباقي |
| تأكيد فعل simple | **inline** داخل بطاقة/زر (نافذة تأكيد صغيرة أعلى الزر) | خفيف بلا كسر السياق |
| معاينة فعل من ws (`card.action_preview`) | **بطاقة داخل thread** — ليست modal | جزء من الحوار، تسمح بالتحرير المتكرر |
| **بند اعتماد** | **page** `queue.approval_detail` | قرار رسمي مدقَّق، يستحق مساراً |
| قائمة الاعتمادات | **page** `queue.approvals` | مركز عمل يومي |
| اقتراح فعل من الدردشة | **بطاقة inline** في thread + drawer end للاستشهادات | تدفق مستمر |
| فتح مصدر استشهاد | **drawer end** بلوحة `panel.citations` — نقر «فتح كامل» ⇒ page سياق المصدر | قراءة موازية |
| Policy Studio simulate | **drawer end** «محاكاة — لا أثر» | يظل السياق التحريري ظاهراً |
| أثر تبعيات (feature/connector) | **modal** بتأكيد typed | قرار حرِج ولا رجعة سريعة |
| تفاصيل صف تدقيق | **drawer end** (سلسلة correlation) — «فتح كامل» ⇒ page | قراءة سريعة عادةً |
| رفع ملف داخل ws | **inline** داخل `ws.composer` + drawer end للتفاصيل بعد الرفع | يظل الحوار مستمراً |
| تعريف/تعديل نوع/فعل/workflow | **page** builder (لا modal) | مهمة طويلة معقّدة |
| diff بين نسختين | **drawer end** جانب التعريف | مقارنة موازية |
| ربط حساب موصل | **modal** خطوات محدودة | حرِج قصير |
| معاينة HITL قبل الموافقة | **modal** بتأكيد typed | قرار حرِج |
| dashboard admin | **page** بمؤشرات دنيا | نقطة انطلاق ثابتة |
| تفصيل تشغيل (runs.detail) — (Δ v0.6) | **page** | مراقبة طويلة تستحق مساراً ورجوع متصفح |
| تفصيل خطوة تشغيل — (Δ v0.6) | **drawer end** (Step Drawer) | قراءة موازية للخط الزمني |
| مخرَج خطوة ضخم — (Δ v0.6) | **offload إلى ملف محكوم** يُعرض من لوحة المخرجات | لا inline dump ولا console |
| تفاصيل مزوّد/نموذج — (Δ v0.6) | **drawer end** | قراءة D9-آمنة بلا حقول حساسة |

## 5) ما يبقى داخل ws.main مقابل ما ينتقل إلى Runtime Renderer
**يبقى في ws.main:**
- المحادثة، الاستيضاح حقلاً-حقلاً داخل `card.action_preview`، بطاقات النتائج ≤10 صفوف، بطاقات الرفض، `card.sql_answer` (النص + الجدول المختصر)، رفع ملف واحد، إنشاء مهمة/بند اعتماد بسيط، فتح مصدر (drawer).
**ينتقل إلى Runtime Renderer (page):**
- أي نتيجة >10 صفوف (زر «عرض الكل في القائمة» ⇒ `run.list` بنفس الفلتر المسلسل)
- تحرير سجل معقّد أو متعدد الأقسام (⇒ `run.form`)
- عرض تفصيلي لسجل مع كل أفعاله من Registry (⇒ `run.detail`)
- Entity Profile كامل بخط زمني (⇒ `profile.entity`)
- تصدير محكوم (زر التصدير يفتح modal ثم يحول إلى page الحالة)
**قاعدة:** بطاقة الدردشة **تفتح** الـ Renderer عبر رابط معلن — لا تُحاكيه.

## 6) ما يبقى داخل Admin Console حصراً
تعريف/تعديل الأنواع (`admin.record_types`)، الحقول، السياسات (RLS/FLS/ABAC/ReBAC/SoD)، عقود الأفعال (`admin.actions`)، تعريف الـ workflows، مصادر المعرفة، ربط القدرات والوكلاء، إدارة الموصلات، Feature Management + Safe Mode، عارضا التدقيق/السجلات، أسطح الإعداد للقراءة (D9). **الوصول من ws.main إلى إعداد إداري = فتح شاشة الـ Console (effect=open) لا تنفيذ.**

## 7) قواعد التنقّل والرجوع
1. **زر رجوع المتصفح** يعمل على كل page/tab — لا تحبس المستخدم.
2. **drawer/modal** يُغلقان بـ Esc + النقر خارجاً (drawer فقط) + زر × ظاهر.
3. لا تفتح **modal تلقائياً** بلا فعل مستخدم صريح — استثناء وحيد: بند اعتماد بانتظار قرار عند فتح `queue.approval_detail` بمعرّف مباشر.
4. **الحفاظ على السياق:** فتح drawer/modal لا يمسح فلاتر القائمة ولا يفقد نص الرسالة قيد الكتابة في ws.composer.
5. **تنقّل داخل ws.main:** فتح شاشة إدارية من بطاقة يفتحها في **تبويب متصفح جديد** (لا يفقد الحوار) — قاعدة إلزامية لأي `effect=open` نحو `/admin/*`.

## 8) الأسطح داخل ws.main (تشريح ثابت)
> **تخطيط الشاشة (منطقياً — RTL):**
> - **أعلى:** topbar (بحث موحّد · جرس · مبدّل ar/en · هوية المستخدم).
> - **بداية-منطقياً:** sidebar مصغّر (Workspace · /me/* · /app حسب الصلاحية).
> - **الوسط (الأكبر):** `ws.thread` — المحادثة + البطاقات + الحالات · تحته `ws.composer` ثابت أسفلياً (نص + إرفاق + Scope selector).
> - **نهاية-منطقياً:** context panel بأربعة تبويبات — Citations · Related · Attachments · Audit (للمخوّل فقط) — واحد فقط مفتوح تلقائياً عند حدث ذي صلة.
- **context panel** تبويبات مطوية عنوانياً: Citations · Related · Attachments · **Audit (للمخوّل فقط)** — واحد فقط مفتوح تلقائياً عند حدث (استشهاد جديد ⇒ Citations؛ سجل مرتبط ⇒ Related).
- **drawer end** يستخدم لمعاينة سجل/مصدر/تدقيق دون مغادرة الحوار.
- **modal** فقط لتأكيدات typed/dual أو HITL.

## 9) قواعد Stitch الملزمة (يجب أن تظهر في كل prompt لاحق)
1. **الافتراضي مطويّ:** كل ما ليس ضرورياً للقرار الحالي يبدأ مطوياً/مخفياً.
2. **سطح واحد لكل غرض:** لا تكرار بين page/drawer/modal لنفس المحتوى.
3. **لا يتحول page إلى modal:** المهام الطويلة صفحات، القرارات القصيرة modals.
4. **لا يُنشأ سطح جديد بلا مبرّر من §4**: إن لم تجد الحالة في المصفوفة ⇒ أضفها بقاعدة، لا ترتجل.
5. **panel.audit مطويّ دائماً**، ولا رأس له لغير المخوّل (لا يوجد شيء يخبره أنه موجود).
6. **drawer end ينفتح ≤400px** ولا يحجب الجدول كلياً؛ **modal ≤640px** ما لم يكن مراجعة قسم تقرير.

## 10) Open Interaction Decisions
| OD | الموضوع | التوصية |
|---|---|---|
| OD-IX-1 | حفظ حالة الـ drawer في URL | لا افتراضياً؛ استثناء: drawer المصدر يحفظ `source_ref` في hash لمشاركة الرابط |
| OD-IX-2 | تبويبات الـ context panel: فتح تلقائي عند وصول محتوى جديد | نعم مع تلميح بصري خفيف؛ يبقى قابلاً للطي الفوري (قابل للتعطيل من تفضيلات المستخدم لاحقاً) |
| OD-IX-3 | زر «فتح كامل» من drawer إلى page | ظاهر دائماً في drawer end بأسفل يمين-منطقي |
