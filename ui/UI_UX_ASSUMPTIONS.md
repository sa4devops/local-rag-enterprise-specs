> **Version:** 1.0 — Proposed (تُعتمد مع حزمة UI/UX كاملة؛ Release مقترح لاحقاً: `v0.4-ui-ux-planning-package`) · **Date:** 2026-07-06
> **الموضع الهدف:** `ui/UI_UX_ASSUMPTIONS.md` في `local-rag-enterprise-specs` · **Authority:** تحت الكتالوج وتصاميم المراحل؛ عند التعارض تحكم وثائق v0.3.
> **قاعدة:** وثيقة تخطيط UX — لا كود، لا CSS، لا تنفيذ. أي بند غير محسوم = Open UX Decision.
> **اعتماد المالك (2026-07-06):** أُقرّ نموذج **Dual-Surface + Action Layer واحد + AI Workspace كواجهة مركزية من P6 + Generated Renderer كسطح حتمي user-facing + نفس action_id من الدردشة والـ Renderer + لا تجاوز صلاحية/تدقيق** — هذه الوثيقة تعكس القرار المعتمد.

## 1) الفلسفة المعتمدة
النظام يُقدَّم للمستخدم كـ **Governed Enterprise AI Command Center**: مساحة عمل ذكية (AI Workspace) هي السطح الأساسي للإنتاجية، تجلس فوق **Action Layer واحد محكوم** (action_id → Permission → Validation → Preview → Confirm → Approval-عند-الحاجة → Execute → Audit). ليست chatbot لأن الدردشة **وسيلة تشغيل** لا مصدر حقيقة؛ وليست low-code builder للمستخدم النهائي لأن بناء الشاشات/الأفعال حِكر على الأدمن عبر الخط المحكوم.

## 2) الافتراضات الحاكمة (A-UX)
| # | الافتراض | الأساس |
|---|---|---|
| A-UX-1 | **Dual-Surface إلزامي:** كل عملية أعمال لها مسار حتمي (شاشة مولّدة/نموذج) ومسار محادثي (Action Card) — كلاهما يستدعي **نفس action_id عبر نفس البوابات**؛ لا منطق خاص بسطح | constitution م2/م3 · P3 FR-3.5 |
| A-UX-2 | **LLM يقترح ولا ينفّذ:** مخرج النموذج = اقتراح action مهيكل يمر تحققاً حتمياً وصلاحية ومعاينة وتأكيداً (وHITL للحساس) قبل أي أثر | م1 · P6 Validation fail-closed |
| A-UX-3 | **Renderer واحد لا عشرات الشاشات:** الشاشات المولّدة تُعرض عبر Runtime Renderer موحّد يلبس أي كيان؛ المصمَّم يدوياً في Stitch هو **القالب** لا كل نوع سجل | P1/P2 Metadata Registry |
| A-UX-4 | **schema الشاشة = عقد البطاقة:** بطاقات المعاينة في الدردشة تُبنى من نفس field_definitions والتحققات — مصدر واحد للحقيقة الواجهية | P2 |
| A-UX-5 | **Phase-aware home:** قبل P6 الصفحة الرئيسية = Work Queue (مهامي/اعتماداتي/إشعاراتي)؛ من P6 تصبح AI Workspace افتراضياً (علَم إعداد لكل دور) | ترتيب المراحل المعتمد |
| A-UX-6 | **Permission-trimmed UI:** الواجهة تعرض المسموح فقط (قوائم/أزرار/حقول)؛ **العرض تلميح والخادم الحكم** — لا قرار صلاحية في المتصفح | م2 · P4 |
| A-UX-7 | **شارات إلزامية:** التصنيف (5 درجات) وعلامة «مُولَّد بالذكاء» وحالة الاعتماد تظهر بصرياً أينما وُجد المحتوى | P4 · P6 FR-6.10 |
| A-UX-8 | **الاستشهاد أو الرفض ظاهران:** كل إجابة أعمال تعرض مصادرها القابلة للفتح، وحالـة «معلومات غير كافية» حالة أولى مصمَّمة لا رسالة خطأ | P5/P6 No-Guessing |
| A-UX-9 | **RTL-first:** عربي أساس/إنجليزي ثانٍ، تخطيط منطقي (start/end)، أرقام وتواريخ محلية، والمعرّفات التقنية ASCII | م6 · م18 |
| A-UX-10 | **حدود D9 بصرياً:** لا شاشة تُظهر endpoints/secrets/مفاتيح؛ أسطح Ops قراءة فقط | ADR-0008/D9 |
| A-UX-11 | **Audit visibility بسياسة:** أثر التدقيق للعنصر (من فعل ماذا) يظهر لمن يملك صلاحية العرض (مدير/أدمن) كلوحة جانبية لا كصفحة سرية | م3 · P1 FR-1.9 |
| A-UX-12 | **مخرجات Stitch تصور بصري فقط:** لا يُعتمد كودها نهائياً؛ التنفيذ لاحقاً React+Vite+TS+Tailwind+shadcn-style وفق D1 وسياسة D8 | D1 · ADR-0021 |
| A-UX-13 | **Desktop-first** (بيئة عمل مؤسسية)؛ استجابة أساسية للأحجام، وتجربة موبايل كاملة Backlog | Backlog |
| A-UX-14 | **اللغة الافتراضية للواجهة عربية** مع تبديل فوري en، وكل النصوص i18n keys | م6 |

## 3) ما يبقى صفحات مستقلة (استثناءات مبررة)
Auth (دخول/أول-دخول/قفل) · **AI Workspace** (P6) · **Work Queue الثلاثية** (مهامي/اعتماداتي/الإشعارات — P3) · **Runtime Screen Renderer** (قائمة/نموذج/تفاصيل) · **Entity Profile** (P2→P4) · **Report Review/Studio** (مراجعة واعتماد رسمية — P6) · **App Store + My Connections** (P7) · **Admin Console بكامله** · **Ops/About** (P8). كل ما عداها يُفضَّل سطحاً داخل الـ Workspace أو لوحة جانبية.

## 4) Non-Goals واجهياً
لا builder مرئي للمستخدم النهائي · لا dashboards تحليلية متقدمة (Backlog/BI) · لا dark-mode في المسار الأول (OD-UX-3) · لا تخصيص ثيمات لكل مستخدم · لا تجربة موبايل أصلية.
