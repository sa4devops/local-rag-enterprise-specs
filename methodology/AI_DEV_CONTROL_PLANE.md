# AI Development Control Plane — قدرة حاكمة (موجز تأسيسي)
> **Version:** 1.0 — Proposed (يُعتمد بتوقيع/دمج SA في بوابة R1-B) · **Date:** 2026-07-21 · **الموضع:** `methodology/AI_DEV_CONTROL_PLANE.md`
> **Authority:** وثيقة قدرة منهجية؛ لا تعدّل ADR-0033 بل تُصالِح معه؛ التنفيذ التجريبي لاحق بقرار مالك (سجل المؤجلات).

## 1. الغرض
نظام محلي مستقل يدير دورة التطوير والمراجعة آلياً بأدنى تدخل بشري، مصدر حقيقته مستودعا specs/platform، ومستقل عن أسماء الأدوات والنماذج.

## 2. المبادئ العشرة (مصالحةً مع ADR-0033)
1) الأدوار والمسؤوليات ومصفوفة اختيار الأدوات/النماذج/المهارات تُعرَّف داخل المستودع (المرجع: `methodology/agent-execution-model.md`).
2) قراءة المرحلة وTask Card وFeature Package والعقود المعتمدة قبل توزيع أي مهمة (‏Context-Pack ‏§16).
3) إسناد التنفيذ لوكيل مناسب ثم تشغيل الفحوص كاملة (بوابات pnpm + اختبارات المرحلة).
4) تصنيف مخرجات أدوات الفحص عبر مصنِّف مستقل: خطأ حقيقي / منطقي-معماري / تعارض قواعد أعمال / تحسين اختياري / إيجابية كاذبة.
5) مراجعة ثانية مستقلة للملاحظات المقبولة ثم تكليف المنفذ بالإصلاح وإعادة الفحوص آلياً.
6) أداة المراجعة لا تعدّل الكود قبل التصنيف والسياسة؛ والمنفذ ليس المراجع الوحيد لعمله.
7) حفظ الأدلة والقرارات والسجلات وhandoffs داخل المستودع أو مخزن أدلة محكوم قابل للتتبع (‏§17).
8) الرجوع للمالك فقط عند: قرار أعمال جديد، تعارض لا تحسمه المواصفات، فشل متكرر، أو الاختبار البشري النهائي.
9) تدخل المستخدم النهائي لاختبار الواجهة والوظائف واعتماد النتيجة، لا لإدارة الخطوات الروتينية.
10) لا اختيار Orchestration نهائياً الآن؛ درجات الأتمتة ومساراتها المحمية والتراجع تحكمها ADR-0033 ‏(L0–L3 · revert-PR) حصراً.

## 3. النطاق الحالي والخطوة التالية
هذه الوثيقة تقنين القدرة والمبادئ فقط. تصميم التجربة الأدنى القابلة للقياس = مرحلة **R8** بقرار مالك (صف سجل المؤجلات)؛ لا تنفيذ ولا اختيار تقنية قبله.
